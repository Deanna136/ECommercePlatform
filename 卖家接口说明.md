## 一、全局说明

### 1.1 API 基础路径
http://localhost:8080/seller

---

### 1.2 鉴权说明

除 **注册接口** 和 **登录接口** 外，所有接口均需在请求头中携带 Token。

| 请求头字段名 | 示例值 | 说明 |
|------------|--------|------|
| `seller_token` | `eyJhbGciOiJIUzI1NiJ9...` | 登录成功后返回，有效期 2 小时 |

> **说明：**
> - Token 存储于 Redis 白名单，登出后立即失效，携带已失效 Token 的请求返回 401
> - 所有需鉴权接口只能操作当前登录卖家自己的数据，无法访问其他卖家数据

---

### 1.3 统一响应格式

所有接口均返回如下 JSON 结构：

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {}
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 状态码，200 为成功，其余为异常 |
| msg | String | 提示信息 |
| data | Object / Array / String / null | 响应数据，无数据时为 null |

---

### 1.4 通用错误码

| code | msg | 触发场景 |
|------|-----|---------|
| 200 | 操作成功 | 请求成功 |
| 400 | 参数校验失败 | 入参为空或格式不合法 |
| 401 | Token 无效 / Token 已过期，请重新登录 | 未登录、Token 失效、已登出 |
| 403 | 无权限访问 | 操作非本人数据 |
| 404 | 资源不存在 | 对象不存在 |
| 500 | 系统繁忙，请稍后再试 | 服务器内部异常 |
| 1001 | 用户不存在 | 账号未注册 |
| 1002 | 密码错误 | 密码不匹配 |
| 1003 | 用户名已存在 | 注册时用户名重复 |
| 1004 | 账号已被锁定/禁用 | 账号被管理员封禁 |
| 1006 | 密码格式错误，长度不能少于6位 | 密码长度不足 |
| 1201 | 卖家不存在 | 卖家账号不存在 |
| 1202 | 店铺名称已存在 | 注册时店铺名重复 |
| 1203 | 店铺类型不合法 | storeCategory 传入非法值 |
| 1204 | 卖家账号已停用，无法操作商品 | 账号被封禁时操作商品 |
| 4001 | 商品不存在 | 商品未找到 |
| 4002 | 商品编号已存在 | 新增商品时 productNo 重复 |
| 4003 | 商品状态不允许操作 | 当前状态不可执行此操作 |
| 4008 | 库存不能小于0 | sku 传入负数 |
| 4009 | 商品价格不能小于0 | price 传入负数 |
| 6001 | 订单不存在 | 订单未找到 |
| 6003 | 订单状态不允许此操作 | 当前状态不可执行此操作 |

---

### 1.5 枚举值说明

**账号状态（status）**

| 值 | 说明 |
|----|------|
| `active` | 正常 |

| `deleted` | 已删除 |

| 值 | 说明 |

```json
{
  "userName": "seller_new",
  "password": "123456",
  "name": "王大卖",
  "idNumber": "310101199001011234",
  "sex": "male",
  "phone": "13900139099",
  "address": "上海市浦东新区",
  "image": "https://img.com/avatar.jpg",
  "storeName": "王大卖的店铺",
  "storeCategory": "clothing"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userName | String | ✅ | 用户名，不能为空，全平台唯一 |
| password | String | ✅ | 密码，不能为空，长度不能少于 6 位 |
| name | String | ❌ | 真实姓名 |
| idNumber | String | ❌ | 身份证号，18位 |
| sex | String | ❌ | 性别：`male` / `female` |
| phone | String | ❌ | 手机号，11位 |
| address | String | ❌ | 地址 |
| image | String | ❌ | 头像图片 URL |
| storeName | String | ✅ | 店铺名称，全平台唯一 |
| storeCategory | String | ✅ | 店铺类型，见枚举值 |

#### 响应示例

**成功（200）**

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": "注册成功"
}
```

**失败示例**

```json
{
  "code": 1003,
  "msg": "用户名已存在",
  "data": null
}
```

```json
{
  "code": 1202,
  "msg": "店铺名称已存在",
  "data": null
}
```

```json
{
  "code": 1203,
  "msg": "店铺类型不合法",
  "data": null
}
```

```json
{
  "code": 1006,
  "msg": "密码格式错误，长度不能少于6位",
  "data": null
}
```

```json
{
  "code": 400,
  "msg": "参数校验失败",
  "data": null
}
```

#### 备注

- `userName` 和 `storeName` 均为全平台唯一，重复时分别返回 1003 和 1202
- 注册成功后账号状态默认为 `active`，可直接登录
- 密码以明文传输，后端负责存储前的安全处理

```json
{
```

```json
  "msg": "用户不存在",
  "data": null
}
```

```json
{
  "code": 1002,
  "msg": "密码错误",
  "data": null

```

```json
{
  "code": 1004,
```

- 登录成功后 Token 写入 Redis 白名单，有效期 2 小时

## 三、商品管理模块

### 3.1 查看商品列表

| 项目 | 内容 |
|------|------|
| **接口路径** | `/seller/product/list` |
| **请求方式** | `GET` |
| **是否需要鉴权** | ✅ 是 |

#### 请求参数

无

#### 响应示例

**成功（200）**

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": [
    {
      "id": 1,
      "productNo": "P001",
      "name": "纯棉T恤",
      "category": "clothing",
      "sku": 100,
      "price": 59.90,
      "image": "https://img.com/p1.jpg",
      "description": "夏季透气纯棉T恤",
      "status": "onsale",
      "salesCount": 120,
      "createTime": "2025-01-01 10:00:00",
      "updateTime": "2025-01-01 10:00:00"
    }
  ]
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 商品 ID |
| productNo | String | 商品编号 |
| name | String | 商品名称 |
| category | String | 商品类型 |
| sku | Long | 库存数量 |
| price | Double | 价格（元） |
| image | String | 商品图片 URL |
| description | String | 商品描述 |
| status | String | 商品状态，见枚举值 |
| salesCount | Integer | 累计销量 |
| createTime | String | 创建时间（yyyy-MM-dd HH:mm:ss） |
| updateTime | String | 最后修改时间 |

**失败示例**

```json
{
  "code": 401,
  "msg": "Token 已过期，请重新登录",
  "data": null
}
```

#### 备注

- **只返回当前登录卖家自己的商品**，不可查看其他卖家商品
- 卖家通过查询 seller_products 表中 seller_id 等于当前登录卖家 ID 的所有数据行，获取对应的 product_id 集合；
- 再根据这些 product_id 遍历商品表，即可查询到该卖家发布的所有商品。
- 返回所有状态的商品（含待审核、已拒绝、已停售等），方便卖家全局管理
- 状态为 `deleted` 的商品不返回
