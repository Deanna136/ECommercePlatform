## 一、全局说明

### 1.1 API 基础路径
http://localhost:8080/buyer

---

### 1.2 鉴权说明

除 **注册接口** 和 **登录接口** 外，所有接口均需在请求头中携带 Token。

| 请求头字段名 | 示例值 | 说明 |
|------------|--------|------|
| `buyer_token` | `eyJhbGciOiJIUzI1NiJ9...` | 登录成功后返回，有效期 2 小时 |

> **说明：**
> - Token 存储于 Redis 白名单，登出后立即失效，携带已失效 Token 的请求返回 401
> - 所有需鉴权接口只能操作当前登录买家自己的数据，无法访问其他买家数据

---

### 1.3 统一响应格式

所有接口均返回如下 JSON 结构：

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {}
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 状态码，200 为成功，其余为异常 |
| msg | String | 提示信息 |
| data | Object / Array / String / null | 响应数据，无数据时为 null |

---

### 1.4 通用错误码

| code | msg | 触发场景 |
|------|-----|---------|
| 200 | 操作成功 | 请求成功 |
| 400 | 参数校验失败 | 入参为空或格式不合法 |
| 401 | Token 无效 / Token 已过期，请重新登录 | 未登录、Token 失效、已登出 |
| 403 | 无权限访问 | 操作非本人数据 |
| 404 | 资源不存在 | 对象不存在 |
| 500 | 系统繁忙，请稍后再试 | 服务器内部异常 |
| 1001 | 用户不存在 | 账号未注册 |
| 1002 | 密码错误 | 密码不匹配 |
| 1003 | 用户名已存在 | 注册时用户名重复 |
| 1004 | 账号已被锁定/禁用 | 账号被管理员封禁 |
| 1006 | 密码格式错误，长度不能少于6位 | 密码长度不足 |
| 1101 | 买家不存在 | 买家账号不存在 |
| 1102 | 身份证格式错误 | idNumber 格式不合法 |
| 1103 | 手机号格式错误 | phone 格式不合法 |
| 4001 | 商品不存在 | 商品未找到 |
| 4004 | 商品已下架 | 商品状态非 onsale |
| 4005 | 商品已被封禁 | 商品状态为 suspend |
| 4007 | 商品库存不足 | sku 不足以满足购买数量 |
| 5001 | 购物车不存在 | 买家购物车未初始化 |
| 5002 | 购物车商品不存在 | cart_items 中无该商品 |
| 5003 | 商品数量必须大于0 | quantity 传入 0 或负数 |
| 6001 | 订单不存在 | 订单未找到 |
| 6003 | 订单状态不允许此操作 | 当前状态不可执行此操作 |
| 6004 | 订单已支付/发货，无法取消 | 退单时订单状态不允许 |

---

### 1.5 枚举值说明

**账号状态（status）**

| 值 | 说明 |
|----|------|
| `active` | 正常 |
| `locked` | 已封禁 |
| `deleted` | 已删除 |

**性别（sex）**

| 值 | 说明 |
|----|------|
| `male` | 男 |
| `female` | 女 |

**商品类型（category）**

| 值 | 说明 |
|----|------|
| `clothing` | 服饰 |
| `food` | 食品 |
| `electronics` | 数码电子 |
| `home_living` | 家居生活 |
| `mother_baby` | 母婴 |
| `sports` | 运动 |
| `books` | 图书 |
| `others` | 其他 |

**商品状态（productStatus）**

| 值 | 说明 |
|----|------|
| `onsale` | 在售（买家可见） |
| `out_of_stock` | 已停售/缺货 |
| `suspend` | 管理员强制下架 |
| `offsale` | 卖家下架商品 |

> 买家端商品列表仅展示 `onsale` 状态的商品

**订单状态（orderStatus）**

| 值 | 说明 |
|----|------|
| `pending` | 待付款 |
| `paid` | 已付款，待发货 |
| `processing` | 处理中 |
| `shipped` | 已发货 |
| `delivered` | 已送达 |
| `completed` | 交易完成 |
| `cancelled` | 已取消 |
| `refunded` | 已退款 |
| `failed` | 交易失败 |
| `abnormal` | 异常（待管理员处理） |

## 二、用户管理模块

### 2.1 买家注册

| 项目 | 内容 |
|------|------|
| **接口路径** | `/buyer/register` |
| **请求方式** | `POST` |
| **是否需要鉴权** | ❌ 否 |

#### 请求体（application/json）

```json
{
  "userName": "buyer_new",
  "password": "123456",
  "name": "李小买",
  "idNumber": "110101199001011234",
  "sex": "female",
  "phone": "13800138099",
  "address": "北京市朝阳区建国路88号",
  "image": "https://img.com/avatar.jpg"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userName | String | ✅ | 用户名，不能为空，全平台唯一 |
| password | String | ✅ | 密码，不能为空，长度不能少于 6 位 |
| name | String | ❌ | 真实姓名 |
| idNumber | String | ❌ | 身份证号，18位格式 |
| sex | String | ❌ | 性别：`male` / `female` |
| phone | String | ❌ | 手机号，11位格式 |
| address | String | ❌ | 收货地址 |
| image | String | ❌ | 头像图片 URL |

#### 响应示例

**成功（200）**

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": "注册成功"
}
```

**失败示例**

```json
{
  "code": 1003,
  "msg": "用户名已存在",
  "data": null
}
```

```json
{
  "code": 1006,
  "msg": "密码格式错误，长度不能少于6位",
  "data": null
}
```

```json
{
  "code": 1102,
  "msg": "身份证格式错误",
  "data": null
}
```

```json
{
  "code": 1103,
  "msg": "手机号格式错误",
  "data": null
}
```

```json
{
  "code": 400,
  "msg": "参数校验失败",
  "data": null
}
```

#### 备注

- `userName` 为全平台唯一，重复时返回 1003
- 注册成功后账号状态默认为 `active`，系统同时为该买家自动创建一个购物车
- 密码长度必须 ≥ 6 位

---

### 2.2 买家登录

| 项目 | 内容 |
|------|------|
| **接口路径** | `/buyer/login` |
| **请求方式** | `POST` |
| **是否需要鉴权** | ❌ 否 |

#### 请求体（application/json）

```json
{
  "userName": "buyer01",
  "password": "123456"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userName | String | ✅ | 用户名，不能为空 |
| password | String | ✅ | 密码，不能为空 |

#### 响应示例

**成功（200）**

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "id": 1,
    "userName": "buyer01",
    "name": "张三",
    "phone": "13800138001",
    "address": "北京市朝阳区",
    "image": "https://img.com/buyer01.jpg",
    "status": "active",
    "token": "eyJhbGciOiJIUzI1NiJ9.eyJpZCI6MSwidXNlck5hbWUiOiJidXllcjAxIn0.xxx"
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 买家 ID |
| userName | String | 用户名 |
| name | String | 真实姓名 |
| phone | String | 手机号 |
| address | String | 收货地址 |
| image | String | 头像 URL |
| status | String | 账号状态 |
| token | String | JWT Token，后续所有请求放入请求头 `buyer_token` |

**失败示例**

```json
{
  "code": 1001,
  "msg": "用户不存在",
  "data": null
}
```

```json
{
  "code": 1002,
  "msg": "密码错误",
  "data": null
}
```

```json
{
  "code": 1004,
  "msg": "账号已被锁定/禁用",
  "data": null
}
```

#### 备注

- 登录成功后 Token 写入 Redis 白名单，有效期 2 小时
- 同一账号重复登录会覆盖旧 Token，旧 Token 立即失效（单点登录）
- 账号被封禁（`locked`）时无法登录，需联系管理员解封

| image | String | ❌ | 头像图片 URL |

> 当前登录买家 ID 由服务端从 Token 中解析获取，无需客户端传递

#### 响应示例

**成功（200）**

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": "个人信息更新成功"

```

**失败示例**

```json
{
  "code": 1102,
  "msg": "身份证格式错误",
  "data": null

```

{
  "code": 1103,

  "data": null

#### 请求参数（Query String）

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|

| maxPrice | Double | ❌ | 最高价格（含） |

> 所有参数均为可选，不传时等同于查询全部在售商品

#### 响应示例

**成功（200）**

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": [
    {
      "id": 1,
      "productNo": "P001",
      "name": "纯棉T恤",
      "category": "clothing",
      "sku": 100,
      "price": 59.90,
      "image": "https://img.com/p1.jpg",
      "storeName": "潮流服饰店",
      "salesCount": 120
    }
  ]
}
```

**失败示例**

```json
{
  "code": 400,
  "msg": "参数校验失败",
  "data": null
}
```

#### 备注

- 多条件 AND 关系，无匹配时返回空数组 `[]`
- **筛选范围仅限 `onsale` 状态的商品**，其他状态商品不参与筛选
- 常用场景：按类型筛选、按价格区间筛选、按店铺搜索
