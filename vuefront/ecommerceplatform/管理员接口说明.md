## 一、全局说明

### 1.1 API 基础路径
http://localhost:8080/admin

---

### 1.2 鉴权说明

除 **登录接口** 外，所有接口均需在请求头中携带 Token。

| 请求头字段名 | 示例值 | 说明 |
|------------|--------|------|
| `admin_token` | `eyJhbGciOiJIUzI1NiJ9...` | 登录成功后返回，有效期 2 小时 |

> **说明：** Token 存储于 Redis 白名单，调用登出或 Token 过期后立即失效，携带已失效 Token 的请求将返回 401。

---

### 1.3 统一响应格式

所有接口均返回如下 JSON 结构：

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {}
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 状态码，200 为成功，其余为异常 |
| msg | String | 提示信息 |
| data | Object / Array / String / null | 响应数据，无数据时为 null |

---

### 1.4 通用错误码

| code | msg | 触发场景 |
|------|-----|---------|
| 200 | 操作成功 | 请求成功 |
| 400 | 参数校验失败 | 入参为空或格式不合法 |
| 401 | Token 无效 / Token 已过期，请重新登录 | 未登录、Token 失效、已登出 |
| 403 | 无权限访问 | 越权操作 |
| 404 | 资源不存在 | 对象不存在 |
| 500 | 系统繁忙，请稍后再试 | 服务器内部异常 |
| 1001 | 用户不存在 | 账号未注册 |
| 1002 | 密码错误 | 密码不匹配 |
| 1004 | 账号已被锁定/禁用 | 账号状态异常 |
| 1006 | 密码格式错误，长度不能少于6位 | 密码长度不足 |
| 1301 | 管理员不存在 | 管理员账号不存在 |
| 2002 | 数据重复/唯一约束冲突 | 数据库唯一约束 |
| 4001 | 商品不存在 | 商品未找到 |
| 4003 | 商品状态不允许操作 | 当前状态不可执行此操作 |
| 6001 | 订单不存在 | 订单未找到 |
| 6003 | 订单状态不允许此操作 | 当前状态不可执行此操作 |

---

### 1.5 枚举值说明

**账号状态（status）**

| 值 | 说明 |
|----|------|
| `active` | 正常 |
| `locked` | 已封禁 |
| `deleted` | 已删除 |

**性别（sex）**

| 值 | 说明 |
|----|------|
| `male` | 男 |
| `female` | 女 |

**商品/店铺类型（category / storeCategory）**

| 值 | 说明 |

| `food` | 食品 |
| `electronics` | 数码电子 |
| `home_living` | 家居生活 |
| `mother_baby` | 母婴 |
| `sports` | 运动 |
| `books` | 图书 |
| `others` | 其他 |

**商品状态（productStatus）**

| 值 | 说明 |
|----|------|
| `offsale` | 下架 |
| `pending_review` | 待审核 |
| `approved` | 审核通过（待上架） |
| `onsale` | 在售 |
| `rejected` | 审核拒绝 |
| `out_of_stock` | 缺货 |
| `suspend` | 强制下架 |
| `deleted` | 已删除 |

**订单状态（orderStatus）**

| 值 | 说明 |
|----|------|
| `pending` | 待付款 |
| `paid` | 已付款 |
| `processing` | 处理中 |
| `shipped` | 已发货 |
| `delivered` | 已送达 |
| `completed` | 已完成 |
| `cancelled` | 已取消 |
| `refunded` | 已退款 |
| `failed` | 交易失败 |
| `abnormal` | 异常 |

## 二、认证模块

### 2.1 管理员登录

| 项目 | 内容 |
|------|------|
| **接口路径** | `/admin/login` |
| **请求方式** | `POST` |
| **是否需要鉴权** | ❌ 否 |

#### 请求体（application/json）

```json
{
  "userName": "admin",
  "password": "123456"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userName | String | ✅ | 用户名，不能为空 |
| password | String | ✅ | 密码，不能为空 |

#### 响应示例

**成功（200）**

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "id": 1,
    "userName": "admin",
    "token": "eyJhbGciOiJIUzI1NiJ9.eyJpZCI6MSwidXNlck5hbWUiOiJhZG1pbiJ9.xxx"
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 管理员 ID |
| userName | String | 用户名 |
| token | String | JWT Token，后续所有请求放入请求头 `admin_token` |

**失败示例**

```json
{
  "code": 1001,
  "msg": "用户不存在",
  "data": null
}
```

```json
{
  "code": 1002,
  "msg": "密码错误",
  "data": null
}
```

#### 备注

- 登录成功后 Token 写入 Redis，有效期 2 小时
- 同一账号重复登录会覆盖旧 Token，旧 Token 立即失效（单点登录）

---

### 2.2 管理员修改密码

| 项目 | 内容 |
|------|------|
| **接口路径** | `/admin/updatePwd` |
| **请求方式** | `POST` |
| **是否需要鉴权** | ✅ 是 |

#### 请求体（application/json）

```json
{
  "id": 1,
  "password": "newpassword123"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | Long | ✅ | 管理员 ID |
| password | String | ✅ | 新密码，长度不能少于 6 位 |

#### 响应示例

**成功（200）**

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": "密码修改成功！"
}
```

**失败示例**

```json
{
  "code": 1006,
  "msg": "密码格式错误，长度不能少于6位",
  "data": null
}
```

```json
{
  "code": 1001,
  "msg": "用户不存在",
  "data": null
}
```

#### 备注

- 密码长度必须 ≥ 6 位，否则返回 1006
- 修改密码后建议重新登录，旧 Token 在过期前仍有效（如需强制下线可在此处同步删除 Redis Token）

### 3.3 买家条件筛选

| 项目 | 内容 |
|------|------|
| **接口路径** | `/admin/buyer/query` |
| **请求方式** | `GET` |
| **是否需要鉴权** | ✅ 是 |

#### 请求参数（Query String）

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | Long | ❌ | 买家 ID，精确匹配 |
| userName | String | ❌ | 用户名，模糊查询 |
| name | String | ❌ | 真实姓名，模糊查询 |
| idNumber | String | ❌ | 身份证号，精确匹配 |
| phone | String | ❌ | 手机号，精确匹配 |
| sex | String | ❌ | 性别：`male` / `female` |
| address | String | ❌ | 地址，模糊查询 |
| status | String | ❌ | 账号状态：`active` / `locked` / `deleted` |

> 所有参数均为可选，不传时等同于查询全部

#### 响应示例

**成功（200）**

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": [
    {
      "id": 6,
      "userName": "buyer06",
      "name": "孙八",
      "sex": "female",
      "phone": "13800138006",
      "address": "成都市高新区",
      "status": "locked",
      "createTime": "2025-01-01 10:00:00",
      "updateTime": "2025-01-01 10:00:00"
    }
  ]
}
```

**失败示例**

```json
{
  "code": 400,
  "msg": "参数校验失败",
  "data": null
}
```

#### 备注

- 多个条件之间为 AND 关系
- 无匹配结果时返回空数组 `[]`，不返回 404

### 3.4 封禁买家账号

| 项目 | 内容 |
|------|------|
| **接口路径** | `/admin/buyer/{id}/ban` |
| **请求方式** | `PUT` |
| **是否需要鉴权** | ✅ 是 |

#### 路径参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | Long | ✅ | 买家 ID |

#### 响应示例

**成功（200）**

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": "封禁成功"
}
```

**失败示例**

```json
{
  "code": 1101,
  "msg": "买家不存在",
  "data": null
}
```

```json
{
  "code": 1004,
  "msg": "账号已被锁定/禁用",

}
```
